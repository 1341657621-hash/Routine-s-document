# Beauty Signal 小红书数据周报 · 采集指南

> 适用于：Beauty Signal 小红书账号（美妆类）
> 更新频率：每周一次（建议每周一采集上周数据）
> 交付格式：PDF（数据可视化周报）

---

## 一、每周需要提供的截图/数据

### 📌 必须项（每周一发送）

#### 1. 账号总览数据
- **路径**：小红书创作服务平台 → 数据看板 → 账号概览
- **需要截图**：
  - 粉丝数据（总粉丝、本周新增、流失）
  - 观看数据（总观看、本周新增观看）
  - 互动数据（点赞、收藏、评论、分享）
  - 互动率（如平台提供）

#### 2. 笔记数据明细
- **路径**：小红书创作服务平台 → 内容管理 → 笔记数据
- **需要截图/导出的数据**：
  - 本周发布的所有笔记列表（标题、发布时间、观看量、点赞、收藏、评论、分享）
  - 本周表现最好的 Top 3 笔记详情
  - 本周表现最差的 Bottom 3 笔记详情

#### 3. 粉丝画像数据
- **路径**：小红书创作服务平台 → 数据看板 → 粉丝数据
- **需要截图**：
  - 粉丝性别分布
  - 粉丝年龄分布
  - 粉丝地域分布（Top 城市）
  - 粉丝兴趣标签
  - 新增粉丝来源（搜索/推荐/关注页等）

#### 4. 笔记互动人群画像
- **路径**：小红书创作服务平台 → 数据看板 → 笔记受众
- **需要截图**：
  - 互动用户性别/年龄分布
  - 互动用户地域分布
  - 互动用户兴趣偏好

### 📌 可选项（有就发，没有不影响）

#### 5. 搜索数据
- **路径**：小红书创作服务平台 → 数据看板 → 搜索数据
- **需要截图**：
  - 搜索来源占比
  - Top 搜索关键词

#### 6. 直播数据（如本周有直播）
- **路径**：直播管理后台
- **需要截图**：观看人数、互动数、新增粉丝

## 二、截图规范

| 要求 | 说明 |
|------|------|
| **清晰度** | 文字可辨认，不模糊 |
| **完整性** | 包含表头、数据行、时间范围 |
| **时间标注** | 截图中需能看到统计周期（如"近7天"、"本周"） |
| **命名规范** | `BeautySignal_YYYYMMDD_数据类型.png` |
| **发送方式** | 直接发到本群即可 |

## 三、周报自动生成流程

```
璐婷发截图 → 我读取数据 → 生成周报 HTML → 导出 PDF → 群内发布
```

**说明：**
- 收到截图后，我会手动提取关键数据填入模板
- 如后续需要自动化，可接入小红书创作服务平台 API（需申请）
- 周报会在每周二/三完成并发布到本群

## 四、周报内容框架预览

### Part 1：本周账号数据总览
- 粉丝增长趋势（环比/同比）
- 观看量 & 互动量总览
- 账号健康度评分（自定义算法）

### Part 2：内容表现分析
- Top 3 / Bottom 3 笔记对比
- 内容类型分布（种草/教程/测评/日常）
- 发布时间 vs 表现关联分析
- 标题风格效果对比

### Part 3：用户画像变化
- 新增粉丝特征（性别/年龄/地域/来源）
- 互动人群 vs 整体粉丝差异
- 兴趣偏好变化趋势

### Part 4：竞品对标
- 选取 3-5 个同类美妆账号
- 粉丝规模/互动率/爆文率对比
- 竞品本周热门内容分析
- 可借鉴方向

### Part 5：下周选题建议
- 基于数据反馈的 5-8 个选题方向
- 每个选题的预估效果 + 执行建议
- 优先级排序（P0/P1/P2）

## 五、FAQ

**Q：如果某周没有发新笔记怎么办？**
A：提供账号总览和粉丝画像即可，内容分析部分会标注"本周无新笔记"，重点分析历史笔记的长尾表现。

**Q：竞品数据怎么获取？**
A：我会通过公开渠道（第三方数据平台、手动采集）获取竞品数据，不需要璐婷额外提供。

**Q：周报可以定制化吗？**
A：可以。第一版交付后，根据反馈调整指标和呈现方式。

**Q：能否接入自动采集？**
A：小红书官方 API 需要申请，目前以截图方式为主。后续如开通 API 权限，可升级为自动采集。
